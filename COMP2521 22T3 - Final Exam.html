Question 11 (15 marks)

Your task is to implement the following function in the file q11/findShortestSublist.c:

List findShortestSublist(List l, int start, int end);

The function takes a list l and two values start and end, and returns a copy of the shortest sublist of l that begins with the value start, and ends with the value end. If there are multiple such sublists with the shortest length, the function should return the first such sublist. If there are no sublists that begin with start and end with end, the function should return an empty list.

Examples

Consider the following list:

50 25 25 12 67 50 9 25 54 67 98 34 9 25 91 67 25 9
  • If start is 25 and end is 50, then the function should return [25, 12, 67, 50].
  • If start is 25 and end is 67, then the function should return [25, 12, 67], because this is the first shortest sublist that begins with 25 and ends with 67 (the others are [25, 54, 67] and [25, 91, 67]).
  • If start is 25 and end is 25, then the function should return [25], because this is the shortest sublist that begins with 25 and ends with 25.
  • If start is 25 and end is 33, then the function should return an empty list, because there are no sublists that begin with 25 and end with 33.

Assumptions and Constraints

  • All general assumptions and constraints at the top of this exam paper apply.
  • The input list must not be modified.
  • You must not use arrays. Failure to comply with this will result in a mark of zero for this question.
  • To get full marks for this question, the time complexity of your solution should be O(n)O(n), where nn is the length of the given list. If your solution is slower than O(n)O(n), the maximum mark you can attain for this question will be 9 (out of 15). (Tip: If you can't think of an O(n)O(n) solution then we recommend that you implement an inefficient solution first.)

Question 13 (20 marks)

Implement the following function in the file q13/closeFollowers.c:

void closeFollowers(Graph g, int src, int distance, int followers[]);

closeFollowers takes four arguments: a directed graph g, a source node src, a distance, and an array followers.

Given a graph g, the function should find all the close followers of src, as defined below. For each close follower i, the function should set followers[i] to 1. For example, followers[2] should be set to 1 if and only if node 2 is a close follower of src. Initially, all values in the followers array are set to -1. followers[i] should remain as -1 if node i is not a close follower of src.

We say a node A is a close follower of node B if the following two conditions are satisfied:
  1. There is a directed path from node A to node B with a path length less than or equal to the given distance, OR A = B (node itself).
  2. For node A, in(A) > 1, where in(A) is the number of incoming edges of node A.

Examples

Consider the following directed graph:

If src is 3 and distance is 2, then:

  • There are three close followers: nodes 1, 3 and 8.
  • Node 0 is not a close follower because there is no path from node 0 to node 3.
  • Node 2 is not a close follower because even though there is a path from node 2 to node 3 with length less than or equal to 2, node 2 only has one incoming edge. For similar reasons, nodes 4 and 7 are also not close followers.
  • Node 5 is not a close follower because even though there is a path from node 5 to node 3, the length of the path is greater than 2 (the given distance). For similar reasons, nodes 6 and 9 are also not close followers.

Assumptions and Constraints

  • All general assumptions and constraints at the top of this exam paper apply.
  • The size of the followers array is equal to the number of vertices in the graph.
  • All values in the followers array are initially -1.
  • To get full marks for this question, you should implement an efficient solution. (The time complexity requirement is intentionally not given - you must judge for yourself whether your solution is efficient enough.) Inefficient solutions will be penalised slightly.
